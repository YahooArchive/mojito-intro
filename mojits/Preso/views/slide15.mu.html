Configuration

  • Configuration can be defined in application.json under specs for an individual mojit or at the root for configuration which is available to the entire application.
  • Configuration can be defined in defaults.json, and definition.json within each mojit.
  • Each configuration file resolves based on the current configuration context.
  • The configuration context must be matched in dimensions.json (both the key and value).
