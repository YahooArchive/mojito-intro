Controllers

  • The controller is the core of a mojit.
  • A controller may run on the client or server (or both).
  • There may be separate controllers for the client and server.
  • Controller methods correspond to actions.
  • Controllers have access to action context (ac), addons, autoloads, models, views, and configuration.

YUI.add('Foo', function(Y, NAME)
{
    Y.mojito.controllers[NAME] =
    {
        init: function(config) {
            this.config = config;
        },
        
        index: function(ac) {
            // do stuff
        }
    };
}, '0.0.1', {requires: ['mojito']});
