Addons and Autoloads

  • Addons are a primary means of sharing code in Mojito and carry an affinity.
  • Addons have access to action context, other addons, autoloads, models, views, and configuration.
  • Autoloads are normal YUI modules and will be deployed automatically if they are required in a binder, controller, etc.
  • Mojito ships with many addons.

YUI.add('FooAddon', function(Y, NAME)
{
    function Addon(command, adapter, ac) {
        // command carries details about the mojit instance.
        // adapter contains done(), error(), etc.
        // ac is the action context of the current mojit.
    }
    
    Addon.prototype = {
        namespace: 'foo',
        doSomething: function () {
        }
    };
    
    Y.mojito.addons.ac.foo = Addon;
}, '0.1.0', {requires: ['FooAutoload']});
